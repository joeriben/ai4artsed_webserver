## Executive Summary

Phase 2+3 combines **prompt input**, **pipeline execution monitoring**, and **media output viewing** into a unified flow experience. Unlike Phase 1 which focuses on discrete selection, Phase 2+3 reveals the **continuous process** of transformation through a box-and-connection visualization that shows users WHERE their prompt sits, WHERE interception happens, and HOW the 4-stage pipeline produces output.

### Why Combined?

**User Mental Model:**
- "I enter a prompt" â†’ "It flows through transformations" â†’ "Output emerges"
- This is ONE continuous experience, not separate "phases"

**Technical Reality:**
- Prompt input determines pipeline flow
- Pipeline execution happens in real-time (observable)
- Output emerges progressively as stages complete
- All three aspects are tightly coupled

**Design Principle:** Show the **journey** from input to output, not just checkpoints.

---

## Visual Philosophy: Pipeline as Flow

### Core Metaphor

**Pipeline = River**
- Input enters at the source
- Flows through 4 stages (like waterfalls/rapids)
- Transformations happen at each stage
- Output emerges at the end
- User observes the journey in real-time

## Real-Time Status Polling

### Polling Strategy

**Implementation:**
```javascript
async startPipelinePolling(runId) {
  this.pollingActive = true;

  while (this.pollingActive) {
    try {
      const status = await api.get(`/api/pipeline/${runId}/status`);

      // Update stage states
      this.updateStageBoxes(status.current_state);

      // Update entity timeline
      this.updateEntityList(status.entities);

      // Update elapsed time
      this.elapsedTime = status.elapsed_time;

      // Stop polling if complete or errored
      if (status.status === 'completed' || status.status === 'error') {
        this.pollingActive = false;
        this.handleCompletion(status);
      }

    } catch (error) {
      console.error('Polling error:', error);
      this.retryCount++;

      if (this.retryCount > 5) {
        this.pollingActive = false;
        this.showError('Lost connection to pipeline. Please refresh.');
      }
    }

    // Poll every 1 second
    await new Promise(resolve => setTimeout(resolve, 1000));
  }
}
```

### UI Updates from Polling

**Stage Box State Mapping:**
```javascript
updateStageBoxes(currentState) {
  const stageNum = currentState.stage;

  // Mark previous stages as complete
  for (let i = 1; i < stageNum; i++) {
    this.stages[i].status = 'completed';
  }

  // Mark current stage as in-progress
  this.stages[stageNum].status = 'in_progress';
  this.stages[stageNum].step = currentState.step;
  this.stages[stageNum].progress = currentState.progress;

  // Mark future stages as pending
  for (let i = stageNum + 1; i <= 4; i++) {
    this.stages[i].status = 'pending';
  }
}
```

## Action Buttons

### Post-Execution Actions

**Primary Actions:**

**1. Generate Again (Same Config)**
- Uses same config, new prompt (or edit existing)
- Fast iteration on variations

**2. Try Different Config**
- Returns to Phase 1
- Preserves current prompt (offer to reuse)

**3. Download Output**
- Downloads media file (PNG/MP3/WAV)
- Filename: `{config_name}_{run_id}.{ext}`

**4. Share**
- Copy link to this run: `/view/{run_id}`
- Generate shareable code (embed run data)
- Social media share (future)

**Secondary Actions:**

**5. View Full Pipeline**
- Expands all stage details
- Shows complete entity timeline
- Export pipeline data as JSON

**6. Export for Research**
- Downloads complete pedagogical journey
- JSON format with all entities + metadata
- For research/analysis purposes

**7. Save to Gallery**
- Saves output to user's personal gallery (future)
- Tag, annotate, organize outputs

## Performance Optimizations

### Lazy Loading Media

**Image Loading:**
```javascript
// Don't load image until Stage 4 completes
if (status.current_state.stage === 4 &&
    status.entities.find(e => e.type === 'output_image' && e.available)) {
  this.loadOutputImage(runId);
}
```

**Progressive Enhancement:**
- Load thumbnail first (smaller size)
- Fade in full resolution
- Use `<img loading="lazy">` for viewport optimization

### Polling Optimization

**Adaptive Polling:**
```javascript
// Poll more frequently during active stages (1s)
// Poll less frequently during slow stages (2s)
const pollInterval = this.isSlowStage(currentStage) ? 2000 : 1000;
```

**Stop Polling When Backgrounded:**
```javascript
document.addEventListener('visibilitychange', () => {
  if (document.hidden) {
    this.pausePolling();
  } else {
    this.resumePolling();
  }
});
```

### Component Rendering

**Avoid Re-Rendering Completed Stages:**
```javascript
// Use Vue's v-once directive for completed stages
<StageBox v-once v-if="stage.status === 'completed'" :stage="stage" />

// Use v-memo for stages that rarely change
<StageBox v-memo="[stage.status, stage.progress]" :stage="stage" />
```

- If execution complete: Return immediately, allow to reselect
