AI4ARTSED USER INTERFACE TECHNICAL REFERENCE FOR TRASHY BOT

PHASE 1: PROPERTY QUADRANTS VIEW (CATEGORY SELECTION)
Interface location is PropertyQuadrantsView.vue at route slash (root). Visual layout shows fixed fullscreen canvas (position fixed, inset 0) with dark background (#0a0a0a). At center viewport is square cluster-wrapper sized at min(70vw, 70vh) maintaining 1:1 aspect ratio. This cluster contains category bubbles arranged in circular formation. Six category bubbles exist, each representing one media/creative domain. Categories are: semantics (speech bubble emoji, blue #2196F3), aesthetics (magic wand emoji, purple #9C27B0), arts (paintbrush emoji, pink #E91E63), heritage (globe emoji, green #4CAF50), freestyle (pointing finger emoji, yellow #FFC107 - positioned at center), technical_imaging (camera emoji, gray #607D8B). The freestyle bubble occupies cluster center (50%, 50%). Other bubbles are distributed evenly on 35% radius circle starting at -45 degrees for X-formation pattern. User clicks/taps any category bubble to select it. Selection logic implements XOR across ALL categories - only ONE category can be active simultaneously. Selecting category A automatically deselects any previously selected category. Visual feedback: selected bubble scales to 1.15x, changes background to category color, displays white border, emits glow shadow. Unselected bubbles show 3px border in category color with 30% opacity, dark background. When category is selected, configuration bubbles (18% width, 1:1 aspect ratio, circular) appear around selected category at 17% distance. Number of config bubbles varies per category. Configs are positioned evenly distributed around parent category using trigonometric calculation starting at -90 degrees (top position). Each config bubble displays preview image background from /config-previews/{{config.id}}.png and text badge overlay at bottom showing localized config name. User clicks config bubble to proceed to Phase 2. Click triggers three operations: (1) emit selectConfig event with config.id, (2) clear pipelineExecution store state, (3) navigate to route /pipeline-execution/:configId. Selected configId is passed via Vue Router params. Config metadata includes: id (string), name (object with en/de keys), description (object with en/de keys), properties (array with single category string), pipeline (string - always text_transformation), icon (emoji string), color (hex string). Phase 2 receives this configId and loads full config plus meta-prompt via API.

PHASE 2: TEXT TRANSFORMATION VIEW (GENERATION INTERFACE)
Interface location is text_transformation.vue at route /pipeline-execution/:configId. Vertically scrolling single-column interface, max-width 1100px, centered. Page has fixed header with return button (left) labeled Phase 1 and title (center). Main content area uses flexbox column layout with responsive gap.

INPUT FIELDS (SIDE BY SIDE BUBBLES)
Input bubble (left) has label Deine Idee: Worum soll es gehen with lightbulb icon. Textarea with 6 rows, bound to inputText ref (empty string initial value). Purpose: User describes creative concept/subject matter. This is primary user idea input. Always editable, no restrictions. When filled, bubble border changes from white low opacity to blue with blue background tint. Context bubble (right) has label Bestimme Regeln, Material, Besonderheiten with clipboard icon. Textarea with 6 rows, bound to contextPrompt ref. Purpose: Contains meta-prompt loaded from backend config context field in user language (German for Youth Flow). This defines creative constraints, style rules, material specifications. Always editable. When user edits, handleContextPromptEdit fires updating pipelineStore.metaPrompt. Initial value loaded via pipelineStore.loadMetaPromptForLanguage during onMounted. When empty, shows required animation (pulsing yellow border), when filled shows blue filled state.

START BUTTON ONE (INTERCEPTION TRIGGER)
Positioned between context bubble and interception preview. Button displays triple arrow Start triple arrow with animated arrows. Enabled condition: inputText has content. Disabled state: opacity 0.3, grayscale filter, pointer-events none. Click action calls runInterception function triggering POST /api/schema/pipeline/stage2 with payload containing schema (configId), input_text, context_prompt, user_language (de), safety_level (youth). No output_config sent - this is pure interception call.

INTERCEPTION RESULT PREVIEW
Label shows Idee plus Regeln equals Prompt with arrow icon. Displays after Start button one click. During loading (isInterceptionLoading equals true), shows large spinner (48px) with blue border-top and explanatory text: Die KI kombiniert jetzt deine Idee mit den Regeln und erstellt einen kreativen Prompt, der zu deinem gewaehlten Kunststil passt. After completion, displays textarea bound to interceptionResult ref containing API response field interception_result (Stage 2 output from pedagogical transformation). Fully editable after loading completes. Manual editing sets executionPhase to interception_done enabling model selection. Visual styling: Green-tinted background, green border 3px solid, green glow shadow.

MODEL SELECTION ACTION
Click triggers selectConfig function which ALWAYS runs optimization even if same model clicked again. Function calls runOptimization making POST /api/schema/pipeline/optimize with payload containing input_text (interceptionResult.value) and output_config (selectedConfig.value). During optimization, model bubble shows loading spinner overlay. Response sets optimizedPrompt ref and executionPhase equals optimization_done.

OPTIMIZED PROMPT PREVIEW
Label shows Modell-Optimierter Prompt with sparkles icon. Appears when selectedConfig has value. During optimization loading, shows spinner with text: Der Prompt wird jetzt fuer das gewaehlte KI-Modell angepasst. Jedes Modell versteht Beschreibungen etwas anders - die KI optimiert den Text fuer die beste Ausgabe. After completion, displays textarea bound to optimizedPrompt ref containing API response field optimized_prompt. Fully editable after loading. Visual styling: Orange-tinted background, orange border 3px solid, orange glow shadow. Empty state: dashed border, no glow.

START BUTTON TWO (GENERATION TRIGGER)
Positioned between optimized prompt and output frame. Same visual style as Start button one. Enabled condition: (executionPhase equals optimization_done OR executionPhase equals generation_done) AND optimizedPrompt.value truthy. Shows safety stamp animation (green checkmark Safety Approved) briefly when generation starts. Click action calls startGeneration then executePipeline.

GENERATION PROCESS
executePipeline function implements intelligent seed logic for iterative correction. Seed determination: (1) If optimizedPrompt unchanged from previousOptimizedPrompt, generate new random seed (user wants variation), (2) If prompt changed, keep current seed (user wants iteration), (3) First run uses default seed 123456789. Progress simulation: generationProgress ref increments randomly every 500ms to 85%, then holds until API response, completes at 100%. API generation call: POST /api/schema/pipeline/execute with payload containing schema (configId), input_text (currentPromptToUse which is optimizedPrompt or interceptionResult or inputText), interception_result (optimizedPrompt.value), context_prompt (contextPrompt.value), user_language (de), safety_level (youth), output_config (selectedConfig.value), seed (currentSeed.value).

OUTPUT FRAME
Width 100%, max-width 1000px, centered. Three visual states: (1) EMPTY - min-height clamp(320px,40vh,450px), dashed border, placeholder text, (2) GENERATING - min-height clamp(320px,40vh,450px), solid green border, animated sprite (SpriteProgressAnimation component receiving generationProgress prop), (3) COMPLETE - no min-height (content determines size), displays media output. Output media types determined by response.data.media_output.media_type. IMAGE displays img tag with click handler for fullscreen modal. VIDEO displays video tag with controls. AUDIO/MUSIC displays audio tag with controls. 3D displays download button with .model file link. Media URL format: /api/media/{{mediaType}}/{{runId}} where runId from response.data.media_output.run_id.

FULLSCREEN IMAGE MODAL
Teleported to body. Fixed position covering viewport (inset 0), black background with blur backdrop-filter. Image max-width 90vw, max-height 90vh, object-fit contain. Close button (times symbol) positioned top-right with white border circle. Click anywhere on modal closes it (sets fullscreenImage ref to null).

EXECUTION PHASE TRACKING
executionPhase ref tracks workflow state: initial (page load) arrow interception_done (after Start button one or manual interception text entry) arrow optimization_done (after model selection) arrow generation_done (after successful media generation). Phase determines button enabled states and scroll behavior.

AUTO-SCROLL BEHAVIOR
Uses scrollDownOnly helper ensuring page only scrolls down, never up. Scroll triggers: (1) After interception - scroll to category section bottom, (2) After category selection - scroll to category section top, (3) After generation start - scroll to pipeline section after 150ms delay. All scrolls use smooth behavior.

USER EDITING CAPABILITIES
All textarea fields (inputText, contextPrompt, interceptionResult, optimizedPrompt) are fully editable at any time after they contain content. User can modify any text and re-run generation with modified prompts. Platform provides suggestions but user has full control. Model selection can be changed at any time to try different models. User can go back to Phase 1 at any time using return button. Platform remembers settings and allows iteration. Seed logic supports both variation (new seed for same prompt) and iteration (same seed for modified prompt).

SPECIAL TECHNICAL NOTES
Model logos stored at /logos/ directory. Config preview images at /config-previews/{{id}}.png. Textarea auto-resize implemented via watch on interceptionResult and optimizedPrompt refs calling autoResizeTextarea which sets height to scrollHeight plus 4px buffer. Session registration occurs after successful generation via updateSession(runId, {{mediaType, configName}}) for chat overlay context.
